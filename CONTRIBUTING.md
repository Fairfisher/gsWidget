# Contributing to gsWidget

First off, thank you for considering contributing to **gsWidget**! It’s people like you who make open-source tools better for everyone.

## 🪲 How to Report Bugs
If you find a bug, please open an [Issue](https://github.com/Fairfisher/gsWidget/issues). To help us fix it faster, please include:
* **Check for duplicates:** Search existing issues to see if the bug has already been reported.
* **Steps to reproduce:** A clear description of how to trigger the error.
* **Environment:** Specify your OS and versions (e.g., Qt version, Compiler, or Python version).
* **Screenshots:** Especially for UI/Widget issues, a visual aid is very helpful.

## 💡 Suggesting Enhancements
Have an idea for a new widget or a better way to do things?
* Open an issue and describe the feature.
* Explain the **use case**: Why is this feature useful for you and others?

## 🛠️ Your First Code Contribution
Ready to write some code? Here is the workflow:

1. **Fork** the repository and create your branch from `main`:
   `git checkout -b feature/your-feature-name`
2. **Coding Style:** Please stick to the existing coding style (indentation, naming conventions).
3. **Examples:** If you add a new widget, please include a small example in the `examples` folder.
4. **Commit Messages:** Use clear, descriptive messages (e.g., `fix: resolved resizing glitch in gsButton`).
5. **Submit PR:** Open a Pull Request and describe your changes. Link any related issues in the description.

## 📜 Code of Conduct
Be kind and respectful to others. We are all here to learn and build something cool together!
